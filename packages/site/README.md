# @profil/site

Implementation starts in M4. Decisions still needed:

- **Framework**: Astro (suited to content sites, has a wikilink plugin
  ecosystem) vs. Jekyll (directly compatible with jyje/profile's Hydejack
  base — minimizes migration cost)
- **Initial knowledge-base tab implementation**: build Quartz separately and
  embed it statically under the `/notes` subpath (fast to start) -> later
  replace with a native renderer (remark-wiki-link + a backlink plugin)
- **Resume tab**: mount the HTML generated by `@profil/jari` directly at a
  route; the download button links to the PDF/DOCX under `dist/`

## Reference

Use jyje/profile's `_layouts/`, `_includes/`, `_sass/` structure as the
homage target — its sidebar cover page and responsive layout patterns are
already proven.
